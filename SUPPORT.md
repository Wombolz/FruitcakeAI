# Support

Use the right channel for the right kind of problem.

## Bug Reports

Open a GitHub issue if:

- FruitcakeAI behaved incorrectly
- a feature regressed
- setup succeeded but runtime behavior is wrong
- the app or backend crashed

Before opening an issue:

1. read the relevant section of [`README.md`](README.md)
2. run `./scripts/doctor.sh`
3. confirm whether you are on the current `main` branch or latest release tag
4. gather logs, screenshots, and exact reproduction steps

## Setup And Install Help

Open a GitHub issue if:

- bootstrap failed
- migrations failed
- models did not download or start correctly
- the client could not connect to the backend

Include:

- your OS and hardware
- the exact command that failed
- the full error output
- whether you changed `.env`

## Security Issues

Do not use public issues for vulnerabilities.

Use the process in [`SECURITY.md`](SECURITY.md).

## Feature Requests

Feature ideas are fine, but the project is not currently open to broad outside co-development.

If you open a feature request:

- keep it narrowly scoped
- explain the use case clearly
- do not assume a pull request will be accepted

The current contribution policy is in [`CONTRIBUTING.md`](CONTRIBUTING.md).

## Pull Requests

General unsolicited code contributions are not the default workflow right now.

If you want to work on something non-trivial, open an issue first and wait for maintainer guidance.
